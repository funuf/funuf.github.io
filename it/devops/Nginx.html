# 简介

Nginx (opens new window) 作为数据传输过程中的一个节点（高性能的HTTP和反向代理服务器），具备提供Web Server、Proxy Server以及LoadBalancer的能力。

Nginx有一个master进程和若干worker进程，master进程负责读取相关配置、维护worker进程。worker进程负责实际处理请求。 如果开启缓存，则缓存加载器及缓存管理进程也会在启动时运行。

Nginx采用基于事件的模型和与操作系统相关的实现机制，以便高效地为worker进程分配请求。（特点是占用内存少，并发能力强）

Nginx的工作方式取决于配置文件，默认的配置文件为nginx.conf，通常位于/usr/local/nginx/conf、/etc/nginx或/usr/local/etc/nginx目录中。

# 命令

操作Nginx的指令很少，如下：

./nginx start
# s-signal
# fast shutdown
./nginx -s stop
# graceful shutdown
./nginx -s quit
# reloading the configuration file
./nginx -s reload
# reopening the log files
./nginx -s reopen
# see version
./nginx -v

重点说一下reload信号：

配置文件发生修改后，不会自动生效，除非使用reload命令或等到nginx重启。

当master进程收到reload配置文件后，它会检查最新的配置文件的内容的语法合法性。

如果验证成功，master会开启新的worker，同时给old worker发送信息，要求他们停止工作。

如果验证失败，master会仍使用之前的配置文件进行工作。

# 配置文件

Nginx的配置文件是模块化的，各模块由特定的指令组成。

指令分为简单指令和块指令。

简单指令包含指令名称和空格分割的参数，并以分号结尾。

块指令与简单指令的结构类似，但不以分号结尾，而是以{}块结尾。

如果块指令内部包含其他指令，则块指令又被成为context。不被任何context包含的指令，称它们位于main context。

例如events和http指令位于main context，server指令在http context，location指令在 server context。

除此之外，以#开头的行成为注释。

为了使配置更易于维护，我们建议您将其拆分为一组存储在 /etc/nginx/conf.d 目录中的特定于功能的文件，在主 nginx.conf 文件中使用 include 指令来引用特定于功能的文件的内容。

# 作用

# 静态内容托管（虚拟主机）

静态内容（即不需要Web服务器进行二次加工即可响应返回的文件（html/js/css/图片））的托管，是Nginx的基础功能，以便资源能够在互联网上共享分发。

http/server(listen/server_name)/location--传送门 (opens new window)

root/index/autoindex/try_files--传送门 (opens new window)

compression--传送门 (opens new window)

# 反向代理（服务托管）

由于具备优秀的高并发特性，Nginx通常会作为反向代理以实现流量聚合和请求分发。

proxy_pass/proxy_set_header/proxy_bind--传送门 (opens new window)

正向代理：代理客户端发送请求

反向代理：代理服务器端接收请求

# 负载均衡（服务分流）

负载均衡是一种分流技术，将大量的客户端请求合理分发至不同的目标服务器，以降低各服务器的负载，保持系统整体的稳定性。

  • 服务端负载均衡 ：Ngnix
  • 客户端负载均衡：OpenFeign（通过Ribbon实现）

upstream/weight/backup/least_conn/ip_hash/hash--传送门 (opens new window)

可选的负载均衡策略有：

  1. Round Robin（支持server weight）
    • 轮询server，同时考虑weight
  2. least connections
    • 代理给当前连接数最少的server
  3. ip hash
    • 基于客户端的ip地址，确定upstream的server
  4. hash
    • 基于客户端信息（ip、port、uri），确定upstream的server
  5. least time（NGINX Plus only）
  6. random（NGINX Plus only）

user  www www;
worker_processes  4;

error_log  /alidata/log/nginx/error.log crit;
pid        /alidata/server/nginx/logs/nginx.pid;

#Specifies the value for maximum file descriptors that can be opened by this process. 
worker_rlimit_nofile 65535;

events 
{
  use epoll;
  worker_connections 65535;
}


http {
	include       mime.types;
	default_type  application/octet-stream;

	#charset  gb2312;

	server_names_hash_bucket_size 128;
	client_header_buffer_size 32k;
	large_client_header_buffers 4 32k;
	client_max_body_size 8m;

	sendfile on;
	tcp_nopush     on;

	keepalive_timeout 60;

	tcp_nodelay on;

	fastcgi_connect_timeout 300;
	fastcgi_send_timeout 300;
	fastcgi_read_timeout 300;
	fastcgi_buffer_size 64k;
	fastcgi_buffers 4 64k;
	fastcgi_busy_buffers_size 128k;
	fastcgi_temp_file_write_size 128k;

	gzip on;
	gzip_min_length  1k;
	gzip_buffers     4 16k;
	gzip_http_version 1.0;
	gzip_comp_level 2;
	gzip_types       text/plain application/x-javascript text/css application/xml;
	gzip_vary on;
	#limit_zone  crawler  $binary_remote_addr  10m;
	log_format '$remote_addr - $remote_user [$time_local] "$request" '
	              '$status $body_bytes_sent "$http_referer" '
	              '"$http_user_agent" "$http_x_forwarded_for"';
	include /alidata/server/nginx/conf/vhosts/*.conf;
}
