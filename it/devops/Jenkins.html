# docker-compose

qcsd安装在nfs服务器上

version: '3.1'
services:
  jenkins:
    image: jenkins:lts
    container_name: jenkins
    restart: always
    ports:
      # 发布端口
      - 8080:8080
      # 基于JNLP的Jenkins代理通过tcp端口50000与Jenkins master进行通信
      - 50000:50000
    environment:
      TZ: Asia/Shanghai
    volumes:
      - ./data:/var/jenkins_home

安装过程会出现数据卷权限问题，用以下命令解决：

chown -R 1000 /usr/local/docker/jenkins/data

  1. 将开发电脑的setting文件上传到Jenkins的volume的对应目录
  2. 在Jenkins中配置settings

  5. 为了构建镜像时的统一操作，需要修改 <output.directory.parent>/var/jenkins_home/jars</output.directory.parent>

# 管理凭据

无论是从仓库拉取代码，还是远程操作其他Linux，它们都需要进行用户认证，因此在Jenkins中提前添加一些凭据，供后续操作使用。

可以添加的凭证有5种：

  1. Username with password：用户名和密码
  2. SSH Username with private key： 使用SSH用户和密钥
  3. Secret ﬁle：需要保密的文本文件，使用时Jenkins会将文件复制到一个临时目录中，再将文件路径设置到一个变量中，等构建结束后，所复制的Secret ﬁle就会被删除。
  4. Secret text：需要保存的一个加密的文本串，如钉钉机器人或Github的api token
  5. Certiﬁcate：通过上传证书文件的方式

# 推送代码触发

有时候，使用git远程仓库无法配置webhook触发jenkins：

  1. 远程仓库不支持，如 阿里云效的代码管理就没有配置webhook的地方
  2. 远程仓库在公网，jenkins在内网，且没有做内网穿透

这时候可以选择jenkins的远程触发

此时推送代码、触发构建可以使用git_push脚本一并完成：

echo 'exec>>> [git push]'
